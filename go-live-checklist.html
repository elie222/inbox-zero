Google Cloud Setup
~30 min
▾

Set up Pub/Sub subscription and verify OAuth scopes for the Chief of Staff webhook.

Verify Google Cloud project exists with Gmail API, Calendar API, and Pub/Sub API enabled
Verify Pub/Sub topic exists (e.g., chief-of-staff-gmail) with gmail-api-push@system.gserviceaccount.com as Publisher
Create a second push subscription on the same topic pointing to the Chief of Staff endpoint: 
gcloud pubsub subscriptions create chief-of-staff-push \
  --topic=chief-of-staff-gmail \
  --push-endpoint="https://YOUR_RAILWAY_URL/api/chief-of-staff/webhook?token=YOUR_TOKEN" \
  --ack-deadline=30
Verify OAuth scopes include gmail.settings.basic (for signatures) and calendar (for shipping events)
Connect nick@smartcollege.com via OAuth (second inbox). Each inbox needs its own OAuth token stored in the Account table.

Slack App Setup
~15 min
▾

Configure Slack interactivity for the approve/edit/reject buttons.

Go to api.slack.com/apps and select your Inbox Zero Slack app
Set Interactivity Request URL to: 
https://YOUR_RAILWAY_URL/api/chief-of-staff/slack/interactions
Create #chief-of-staff channel (private recommended) and invite the bot
Copy the channel ID (right-click channel name → Copy link → ID is in the URL, starts with C)

Acuity API Setup
~5 min
▾

Get your Acuity REST API credentials.

Log into Acuity Scheduling → Integrations → API
Copy User ID and API Key — you'll need both for the .env file

Environment Variables
~5 min
▾

Add Chief of Staff variables to your .env (or Railway dashboard).

Set Acuity credentials: 
ACUITY_USER_ID=your_user_id
ACUITY_API_KEY=your_api_key
Set Slack channel ID: 
CHIEF_OF_STAFF_SLACK_CHANNEL_ID=C0xxxxxxxxx
Generate and set cron secret: 
CRON_SECRET=$(openssl rand -hex 32)
Verify existing vars are set: ANTHROPIC_API_KEY, SLACK_SIGNING_SECRET, GOOGLE_PUBSUB_VERIFICATION_TOKEN, DATABASE_URL

Set Up Cron Jobs
~10 min
▾

Create scheduled HTTP requests that hit the cron endpoint. Use Railway cron, cron-job.org, or similar.

Every request needs header: Authorization: Bearer YOUR_CRON_SECRET
Retry failed — every 5 min: 
GET /api/chief-of-staff/cron?type=retry-failed
Batch summary — every 4 hours: 
GET /api/chief-of-staff/cron?type=batch-summary
Refresh signatures — daily at 6am CT: 
GET /api/chief-of-staff/cron?type=refresh-signatures
Refresh VIP cache — daily at 6am CT: 
GET /api/chief-of-staff/cron?type=refresh-vip-cache
