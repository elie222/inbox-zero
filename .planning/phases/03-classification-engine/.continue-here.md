## Critical Anti-Patterns (do NOT repeat these)

- **ANTI-PATTERN: Trusting "RESOLVED" notes in STATE.md without verifying observable state.** STATE.md said the ECONOMY env vars were resolved 2026-04-27. They were set in SSM but never propagated to the container. Mitigation: when verifying a fix, check the *delivered* artifact (env in container, logs from running process), not the *source* (SSM, repo).
- **ANTI-PATTERN: Partial code patches across forked code paths.** The Phase 3 "Haiku-only" commit only patched the single-rule branch of ai-choose-rule.ts; the multi-rule branch still routed to Sonnet via inherited modelType. Mitigation: when fixing a model selection, grep all `getModel(` and `modelType` call sites in the affected feature path.
- **ANTI-PATTERN: Production config files living only on the server.** docker-compose.yml drifted because there was no repo copy. Mitigation: any prod config change goes back to the repo in the same session.
