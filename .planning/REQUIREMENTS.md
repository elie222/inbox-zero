# Requirements: Personal Email AI

**Defined:** 2026-04-27
**Core Value:** Inbox only shows things that need Rebekah — everything else is already filed before she opens Gmail.

## v1 Requirements

- [ ] **OPS-01**: Daily digest sends successfully from inbox-digest@tdfurn.com (currently fails — wrong from domain)
- [ ] **OPS-02**: Signups are locked to rebekah@trueocean.com — no other accounts can be created
- [ ] **OPS-03**: GitHub Actions CI/CD builds and pushes Docker image to ghcr.io/rebekah-create/inbox-zero-rebekah on push to main
- [ ] **OPS-04**: Server docker-compose.yml pulls from fork image (not upstream elie222 image)

### Recon (RECON)

- [x] **RECON-02**: Inbox Zero's rules engine is mapped — how rules are stored, evaluated, and applied
- [x] **RECON-03**: Inbox Zero's AI integration is mapped — which models, which endpoints, what prompts
- [x] **RECON-04**: Database schema is mapped — key tables relevant to classification and digests
- [x] **RECON-05**: Each major component has a keep/replace/extend decision with rationale
- [x] **RECON-06**: Cost analysis of current Inbox Zero AI usage vs. proposed three-tier architecture

### Classification (CLASS)

- [x] **CLASS-01**: Every incoming email is classified into exactly one of N categories. Verified 2026-05-04 against final production matrix: Marketing, Receipts, Newsletters, Urgent, Uncertain, Internal, 2FA, TD Furn (Deals + Greers List dropped via user curation; Internal + TD Furn added).
- [x] **CLASS-02**: Classification uses three-tier pipeline — rules engine first (free), then Haiku for uncertain, then Sonnet for hard cases only. Verified 2026-05-04: 100% Haiku post cost-fix deploy, 0 Sonnet escalations, 0 fallback warnings.
- [x] **CLASS-03**: Each classification includes a confidence score stored in Postgres (AI tier only; null for STATIC/LEARNED_PATTERN matches per design). Verified 2026-05-04 via 14d organic data.
- [x] **CLASS-04**: Urgent and Uncertain emails stay in Gmail inbox; all others are labeled and archived. Verified 2026-05-04 — except Receipts (intentional user edit: LABEL+DIGEST, no ARCHIVE).
- [ ] **CLASS-05**: 2FA/OTP emails are auto-deleted after a delay. **Deferred** to Phase 999.2 — DELETE ActionType not in upstream schema; spec changed from 24h to ≥1h. Current 2FA rule is LABEL+ARCHIVE.
- [x] **CLASS-06**: Static-rule path bypasses AI for matching emails (originally specified for greers@trueocean.com; Greers List rule was dropped, but the static/learned-pattern code path is exercised by Group-based learned matching — 78 executions in 14d with no AI call).
- [x] **CLASS-07**: Explicit user rules are applied with conversation meta-rules disabled. All 4 conversation rules (To Reply, Awaiting Reply, FYI, Actioned) are `enabled=false`; meta-rule guard from 03-04 in place for Phase 5.
- [x] **CLASS-08**: Classification pipeline processes incoming emails within 2 minutes. Verified 2026-05-04: median ~2.5s, 50× headroom under SLA.

### Daily Digest (DIGEST)

- [ ] **DIGEST-01**: Daily digest email is sent to rebekah@trueocean.com at 9:00 AM ET, covering all DigestItem rows since the last successful send (no upper window cap).
- [ ] **DIGEST-02**: Digest contains Urgent items section with sender, subject, one-line summary
- [ ] **DIGEST-03**: Promotional items in the Marketing section appear as a Sonnet-detected sub-cluster with labels prefixed "Deals — {topic}". No separate top-level Deals section. No separate classification rule (per Phase 3 D-05 and Phase 4 D-18).
- [ ] **DIGEST-04**: Digest contains auto-filed summary counts by category (e.g., "14 newsletters, 6 receipts filed")
- [ ] **DIGEST-05**: Digest contains Uncertain items section. Each Uncertain item has a single "Review in app →" deep-link to https://inbox.tdfurn.com/uncertain/{itemId}. Rich feedback handling (thumbs-up/down, narrative form) is deferred to Phase 5/6.
- [ ] **DIGEST-06**: Digest narrative summary is generated by Claude Sonnet (once per day — acceptable cost)
- [ ] **DIGEST-07**: ~~Clicking thumbs-up/down in digest email records feedback without requiring login~~ — **Descoped from Phase 4 to Phase 6 Feedback System** (per Phase 4 D-10/CONTEXT.md). Email-side feedback ships under FEEDBACK-01.

### Rules Management (RULES)

- [ ] **RULES-01**: A web page exists at inbox.tdfurn.com/rules for managing explicit classification instructions
- [ ] **RULES-02**: Rebekah can add a rule in plain language (e.g., "emails from xyz about abc → Urgent")
- [ ] **RULES-03**: Rebekah can specify sender-only rules (e.g., "only show transactional emails from company X")
- [ ] **RULES-04**: Rules can be edited or deleted from the page
- [ ] **RULES-05**: Active rules are serialized into the classification prompt at runtime
- [ ] **RULES-06**: Rules page requires no login (single-tenant, internal access only)

### Feedback System (FEEDBACK)

- [ ] **FEEDBACK-01**: Thumbs-up/down clicks in digest email are recorded in Postgres with email ID and classification
- [ ] **FEEDBACK-02**: Digest contains a link to a narrative feedback form for detailed corrections
- [ ] **FEEDBACK-03**: Narrative feedback form accepts free-text input and associates it with the day's digest
- [ ] **FEEDBACK-04**: Gmail label changes (relabeling an email) are detected and stored as classification correction signals
- [ ] **FEEDBACK-05**: Gmail deletes of auto-filed emails are detected and stored as negative classification signals
- [ ] **FEEDBACK-06**: Accumulated feedback is included in the classification prompt context to improve future classifications

### Backlog Triage (BACKLOG)

- [ ] **BACKLOG-01**: System can process the existing 100k+ email backlog using the three-tier classification pipeline
- [ ] **BACKLOG-02**: Backlog processing proposes bulk actions (delete, label, archive) grouped by category and sender
- [ ] **BACKLOG-03**: Rebekah reviews and approves/rejects batch proposals before any action is taken in Gmail
- [ ] **BACKLOG-04**: Backlog processing is rate-limited to respect Gmail API quotas and stay within AI cost ceiling
- [ ] **BACKLOG-05**: Backlog triage uses Haiku Batch API to minimize cost (~$10-15 one-time estimate)

## v2 Requirements

### Learning & Graduation

- **LEARN-01**: Classification patterns with high confidence and positive feedback are surfaced for Rebekah to promote to permanent Gmail filters
- **LEARN-02**: Classification prompt is regenerated periodically from full feedback history
- **LEARN-03**: Low-confidence patterns are flagged for more Uncertain classification until enough feedback accumulates

### Deal Intelligence

- **DEAL-01**: Per-sender deal thresholds (e.g., Harbor Freight ≥20% discount, Home Depot power tools only)
- **DEAL-02**: Deal threshold rules configurable from rules page

### Monitoring

- **MON-01**: Simple dashboard showing classification stats, feedback trends, AI cost per day
- **MON-02**: Alert if daily AI cost exceeds $0.50 (anomaly detection)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Multi-user support | Single-tenant personal system — unnecessary complexity |
| Mobile app / push notifications | Morning digest covers the use case |
| Real-time inbox chat | Not an email management problem |
| OAuth login for rules page | Single-tenant — no auth needed |
| Local Redis | Upstash works fine, not worth switching |
| Automatic Gmail filter graduation | Needs confidence data first — v2 |
| Any AI processing over $10/mo additional | Hard cost ceiling — architecture must respect this |
| Upstream Inbox Zero features we don't use | Keep fork minimal to ease upstream merges |
